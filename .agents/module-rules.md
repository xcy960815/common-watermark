# 模块规范

## 公共 API

- `src/core/watermark.ts` 是 Rollup 库入口；其导出构成公开 API，消费者只能依赖包公开导出。
- 公共 API 变更后，运行构建生成 `types/index.d.ts`，检查声明差异，并同步更新 README 和示例。
- 示例必须以消费者方式使用公开 API，不能通过相对路径引用 `src/`。

## DOM 与生命周期

- 水印实例必须按目标元素隔离；不能依赖固定全局 DOM ID、共享清理状态或覆盖全局事件处理器。
- 新增的监听器、观察器、计时器和临时 DOM 节点都必须在 `removeWatermark` 路径中清理。
- 不得破坏宿主事件、可访问性语义、交互能力或已有内联样式；如确需临时调整，必须能够恢复并按决策门槛确认。
- Canvas 内容、透明度、层级、布局与刷新策略都属于消费者可见行为，不在未确认时改变。

## 类型与兼容性

- 保持浏览器运行时边界：不要在模块顶层读取 `window` 或 `document`，除非兼容性决策已确认。
- 新增浏览器 API 前，明确兼容策略、降级路径和清理语义。
- 不以 `any` 或不安全断言掩盖空值、生命周期或兼容性问题。
